4月1日考试

Posted on 2018-04-01 | In 信息学竞赛 , 考试 |
人生苦短, 考试真累.

  • 总体感觉：智商真是捉急
  • 会做两个题, 但是都因为各种原因挂了, 虽然最后都调出来了.
  • 没有更大的测试数据这是一个大问题, 因为我也不会再写个暴力去对拍.暴力通常情况下比正解难写.

智乃

情况

　　 开始怎么都推不出样例来, 感觉"redotopc"和"odcpoter"是可以相互转换的, 然后开心的好像发现了新大陆, 然后找出是怎么转换的给lyq看, 然而他遗憾的告诉我, 这个确实是可以相互转换的, 不能相互转换的是另一组"dog"和"god", woc, 感情我原来是个智障, 此时距考试结束还有15分钟, 恩, 然后改了改, 交上去得了0分, 智商真是感动, 现在还是不会做.
　　我是推了一个错误的性质并且还写了一个hash的类, 码量惊人, 大概一共花在这个题目上2个半小时, 这也间接导致了后面的题目崩溃.

solution

　　这个题需要去发现一些字符串的性质, 这正是我所不擅长的.

  • 因为只能翻转偶数的前缀, 所以如果是奇数位的其最后一个字符肯定不会被翻转.
  • 可以发现相邻的两个字符无论怎么翻转都是连续的, 因此就比较 interesting.

根据上述性质就可以乱搞过了.

code

这是我写的, 当然是错误的那个.

麻耶

solution

　　听说正解是$o(n\log n)$的复杂度, 然而我是$o(n)$, 美滋滋.
　　一眼的贪心, 然后随便码了码, 恩, 就没管, 然后只得到了10分, 考完试后发现答案已经超过了 long long , 然后随便改了改就通过了.

code

惠

solution

　　一眼的模拟题, 然后随便码了码花了大概20分钟就码完了（听说有人考试时一直在码这个题然后还码了两份不同的代码, 真是崇拜, 并且都通过了）, 然后就不管了, 然后只得了10分, 考完看了下数据, 真是感动, 最后一种操作输错顺序了, 题目说是按创建的先后顺序给出, 然后向来不认真看题的我又没看见, 然后随便改了改, 就通过了.
