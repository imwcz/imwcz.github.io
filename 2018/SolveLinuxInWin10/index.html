win10 Linux子系统初体验

Posted on 2018-07-11 | In 解决方案 , Linux |

自认为用过一点点Linux, 然而我不能不用Windows, 当然也需要用一点Linux.

以前尝试过一些能同时用Window和Linux的解决方案, 但是win10提供的这个Linux解决方案真是非常优秀, 比啥双系统, 虚拟机和再买一台电脑的解决方案都好多了.体现在这样几个方面

  • 无缝衔接（终端）
  • 体积小（如果不装图形化界面）
  • 效率高（相对于虚拟机）

Bash

装上Linux子系统之后终于拥有一个像模像样的终端啦!开心呢.
而且还可以装上Vim之后尽情写写代码啦!
可以一边写着, 一边直接编译运行, 写代码简直太舒服了.

图形化界面

详细请见Win10子系统安装图形化界面的两种方法

两种方式

  • 远程桌面连接
  • Xming

感觉前者能干的完全被后者包括了~后者包括一个全屏模式.~

但是安装桌面好麻烦啊!好几个G的空间啊!
还有各种依赖关系安装各种各种东西.

最重要的一点: Linux的图形化界面比Windows落后了至少10年以上
那我用它的图形化界面干啥呢?

软件

~有些软件需要图形化界面的配置~
有时候一些在win上无法使用的软件就可以在Linux上安装了. 这种软件几乎没有吧.例如:

  • sudo apt-get install monodevelop
  • DISPALY=:0 monodevelop

话说这个IDE有Win版
就打开了一个Linux风格的窗口, 然后就尽情享用吧!

总结

我还是感觉我的Win10好用一些.
然而各有利弊还是取其长弃之短好了.

功能 Win10 Linux
开发 + -
娱乐 + -
学习 + -
工作 + -
装B - +
性能 - +
服务器 - +
稳定性 + -

所以最后得出一个结论:

装B的时候用Linux, 其他时候用Win10.
