第一题

暴力

用一个map, hash_table存$i^2, i = 1, 2, \cdots, n$.然后查找, 也可以直接二分查找不用存.

考试的时候感觉时间挺紧的就特判了一下p=q的情况, 还判错了.我真菜.
60分

正解

通过打表找规律，可以发现答案和p，q模4的余数有关系。
如果p=q：
1. p=2，ans=4
2. P%4=1,ans=12
3. P%4=3,ans=4

如果p不等于q：
1. p=2,q%4=1,ans=8
2. P=2,q%4=3,ans=0
3. p%4=1,q%4=1,ans=16
4. P%4=1,q%4=3,ans=0
5. P%4=3,q%4=3,ans=0
这个结论可以用数学方法证明正确性，但比较复杂。

下次遇到这种题(出题人)直接无视顺便蒽娣觞掫燚憞.

第二题

这套题中唯一会做的题, 比标算复杂度少个$\log$.

解法

对于第一问, 在深度优先遍历中维护一个单调下降的栈, 因为节点的子树会改变栈, 但是每次只会改变栈中一个元素, 改回来就好了, 参考P3994 高速公路的做法.

至于第二问……节点的答案可以直接利用父亲节点的答案.

标算

倍增, 维护一个节点下一个比他小的位置.

第三题

20分

40分

线段树维护

100分

将每个格子视作点。将同色相邻点连边。先考虑没有环时（原图田字形也算一个环），记a[i]为单看第i列时的颜色段数，b[i]为第i列与第i-1列有多少个位置颜色相同。l~r列答案即为( a[l]+a[l+1]+…+a[r] ) - ( b[l+1]+b[l+2]+…+b[r] )。
考虑每新出现一个环 ，上述方法计算的答案就会比正确答案小1，于是我们只需计算l~r列完整的环的个数，也就是计算每个极小环（只内部不再嵌套其他环）的左右端点。
知道每个极小环的左右端点后，可以使用二维前缀和或者区间dp的方式，每次O(1) 计算l~r列完整的环的个数。
区间Dp：f[i][j]=f[i+1][j]+f[i][j-1]-f[i+1][j-1]+a[i][j]
有两种方法可以统计每个极小环的左右端点。

方法一 
用高级数据结构维护。
方法二
以原图内部格点为点（下图红色点），在相邻格数字不同时连边（每条黄色边两侧数字均不同），新图中每一个不与外边界联通的独立连通块均表示一个极小环。使用并查集维护每个连通块左右最值即可求出每个极小环左右端点。O( mn + m^2 +  q ) ，期望得分100分。
