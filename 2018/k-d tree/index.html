k-d tree

Posted on 2018-05-22 | In 信息学竞赛 , 总结 |

引入

　　身边的lgj同学前几天看了这个东西, 我看这个东西还蛮有用的就看看.然后其实挺简单有效的.哦,对了, 我写的k-d tree几乎完全来自自己意淫.所以复杂度和常数无法保证.还有我的二叉搜索树是在写k-d tree之前几个小时意淫的, 所以k-d tree和二叉搜索树完全一个写法.

What is kd tree?

　　In computer science, a k-d tree (short for k-dimensional tree) is a space-partitioning data structure for organizing points in a k-dimensional space. k-d trees are a useful data structure for several applications, such as searches involving a multidimensional search key (e.g. range searches and nearest neighbor searches).

二叉搜索树

　　二叉搜索树可以看做是 k-d tree 在k=1时的特例.考虑二叉搜索树划分的过程.树的每个节点都有一个一元数据, 首先选择一个点作为树的根, 将小于根的键值放置在根的左子树, 将大于根的键值放到根的右子树.这样就相当于是将键值映射到一个坐标轴上, 以根代表的数据为分界线将坐标轴划分为两部分.

这就是一个二叉搜索树的实例.事实证明二叉搜索树在随机数据下表现还是不错的.而构造的能使树退化为链的数据就比较慢了.

　　因为k-d tree可以说是一种二叉搜索树的衍生产品, 所以无论是写法还是原理都非常相似, 建议先学会二叉搜索树并且会基本的操作.

k-d tree

　　类比二叉搜索树, k-d 实际上也是一个不断划分的过程.不同的是它的键值是多元的, 而我们只能比较一个整数的大小关系.所以我们可以这么做, 选择数据的一维, 将所有键值根据这一维的大小划分为两堆, 递归下去就构造出了一棵二叉树.就像下图

如何构造kd tree

　　下面用一个平面来举例说明kd tree的构造.

  1. 确定一个点root, 将平面划分为两部分.
  2. 在左边的点中确定一个点, 将左边划分为两部分.
  3. 在右边的点中确定一个点, 将右边划分为两部分.
  4. ……


随后我们便得到了一个树形结构.这便是我们构造出来的kd tree.
　　那么该如何选择键值的哪一维进行比较以及选择哪个点作为root呢呢?常见的做法是选择方差最大的那一维, root选择中位数.

如何在某一维上比较坐标

　　定义一个比较函数或者是重载运算符.比较一个树上的节点和一个坐标我使用了Compare(Node *,Point)

  • 当坐标完全相同也就是为同一个点时返回false;
  • 当某一维$a_i<b_i$时返回1;
  • 当某一维$a_i\ge b_i$时返回-1;

如何插入节点

　　遍历到所在的最小子空间, 然后新建一个节点即可.

如何查找平面上点是否存在

　　像二叉搜索树的步骤一样, 从kd tree的root开始遍历树.例如, 判断$(6,7)$是否存在?

  • 处在$\text{A}$,$6\le A_x$, 进入左边
  • 处在$\text{B}$,$7\ge B_y$, 进入上面
  • 处在$\text{D}$,$6\ge D_x$, 进入右边
  • 处在$\text{I}$,$7\ge I_y$, 进入上面
  • 进入空节点.未找到

查询最近点

　　这里的距离一般指的是欧几里得距离.
　　就按照上面的例子, 查找距离$(6,7)$最近的点.我们可以这么做:

  • $(6,7)$位于$\text I=(6,5)$的子空间里, 最小距离为4.
  • 回溯至$\text{D}$, 更新最小距离为1, $(6,7)$到直线$x=5$的距离为1, 不需要进入$\text{H}$
  • 回溯至$\text{B}$, $(6,7)$到$y=4$距离大于1, 不需要进入$\text{E}$.
  • 回溯至$\text{A}$, $(6,7)$到$x=8$距离大于1, 不需要进入$\text{C}$.
  • $\text{A}$是k-d tree根节点, 无法回溯.

查询最近k个点

　　与查询最近点其实思路类似.

例题

BZOJ 2648: SJY摆棋子

Description

这天，SJY显得无聊。在家自己玩。在一个棋盘上，有N个黑色棋子。他每次要么放到棋盘上一个黑色棋子，要么放上一个白色棋子，如果是白色棋子，他会找出距离这个白色棋子最近的黑色棋子。此处的距离是 曼哈顿距离 即(|x1-x2|+|y1-y2|) 。现在给出N<=500000个初始棋子。和M<=500000个操作。对于每个白色棋子，输出距离这个白色棋子最近的黑色棋子的距离。同一个格子可能有多个棋子。

Solution

　　实际上有两种操作:

  • 往平面上加入一个点.
  • 查询平面上距离一个点最近的点.

　　这两种操作完全可以由k-d tree完成.第一种操作只需要遍历k-d tree找到空位插入即可.第二种操作是查询最近点, 不过是将原来的欧几里得距离变为了曼哈顿距离, 问题的实质没有改变.那么这个题就是一个k-d模板题.
