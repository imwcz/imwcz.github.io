此文章内容来自《Pro Git》前两章.

起步

关于版本控制(VCS)

本地版本控制系统

保存并管理文件补丁

集中化的版本控制系统(CVCC)

有一个单一的集中管理的服务器, 保存所有文件的修订版本.给客户端提供文件快照.
但是当中央服务器出故障时, 就无法工作.甚至丢失数据.

分布式版本控制系统

客户端将代码仓库完整的复制下来.

Git

Linux开源社区开发.

差异

  • Git关心数据整体的变化, 其他系统关心具体差异.它保存每次更新时的文件快照.
  • 几乎所有操作都可在本地执行.
  • 时刻保持数据的完整性.在保存之前, 所有数据都要进行内容的校验和计算.并作为数据的唯一标识和索引.作为设计哲学.
    他使用 SHA-1 算法计算数据的检验和.将hash值作为索引, 不是文件名.
  • 一旦提交快照就不用担心丢失数据.

    文件的状态

  1. 已提交, 文件已被保存于本地数据库.
  2. 已修改, 已修改但未保存.
  3. 已暂存, 将已修改的文件放在下次提交的清单里.

    配置

    git提供了一个git config的工具, 用来配置或读取相应的工作环境变量.这些环境变量决定git的工作方式和行为.他们存在于:
  • /etc/gitconfig文件.对系统所有用户普遍适用.–system.git安装目录.
  • /.gitconfig只适用于该用户.–global.存放在用户根目录.
  • 工作目录.git/config文件, 针对当前项目
    每个级别的配置会覆盖上层的相同配置.

    用户信息

    个人的用户名称和电子邮件地址.

每次提交都会引用这两条信息, 随更新内容一起永久纳入历史记录.
去掉–global选项会保存在当前项目.

记录更新到仓库

两种状态(跟踪于未跟踪)

  • 跟踪, 被纳入版本控制管理的文件
    add命令开始跟踪.文件是跟踪文件, 目录是跟踪目录下的文件.

运行了add之后又修改的文件需要再次将最近版本暂存.

忽略某些文件

有些文件无需纳入Git管理, 不希望出现在未跟踪文件列表.
新建一个.gitignore的文件, 列出忽略的文件模式.

  • *.cpp表示忽略cpp文件
  • *~忽略所有以~结尾的文件
  • !sad.cpp这个文件例外
  • /TODO 忽略TODO文件不包括子目录
  • build/忽略目录下所有文件
  • doc/*.txt不包括子目录

使用

移除文件

  • 从已跟踪文件清单中移除(从暂存区域移除), 使用git rrm
    这样会从工作目录中删除制定的文件, 不会出现在为跟踪文件清单中了.
  • 如果只是从工作目录中删除文件, 则会变成Changed but not updated状态.再次运行git rm记录此操作.
  • 如果删除之前修改过并且已经暂存的话, 用-f选项.
  • 如果移除跟踪但不删除文件, 用--cached选项.

查看提交历史

  • -p选项展开每次提交的内容差异
  • -2近显示最近两次更新
  • --start仅简要的显示增减改行数统计
  • --pretty=oneline每个提交放在一行显示
    有short,full,fuller可以用

  • format可以定制要显示的格式

    选项 说明
    %H 提交对象（commit）的完整哈希字串
    %h 提交对象的简短哈希字串
    %T 树对象（tree）的完整哈希字串
    %t 树对象的简短哈希字串
    %P 父对象（parent）的完整哈希字串
    %p 父对象的简短哈希字串
    %an 作者（author）的名字
    %ae 作者的电子邮件地址
    %ad 作者修订日期（可以用 -date= 选项定制格式） %ar 作者修订日期，按多久以前的方式显示
    %cn 提交者(committer)的名字
    %ce 提交者的电子邮件地址
    %cd 提交日期
    %cr 提交日期，按多久以前的方式显示
    %s 提交说明
    提交者和作者有差别
    作者指实际做出修改的人, 提交者是指将此工作成果提交的人.

  • 用oneline或者format时结合--graph选项, 展示每个提交的分支和分化衍合情况.

    限制输出

    -(n) 仅显示最近的 n 条提交
    –since, –after 仅显示指定时间之后的提交。 –until, –before 仅显示指定时间之前的提交。 –author 仅显示指定作者相关的提交。 –committer 仅显示指定提交者相关的提交。

    你可以给出各种时间格式，比如说具体的某一天（“2008-01-15”），或者是多久以前 （“2 years 1 day 3 minutes ago”）。 还可以给出若干搜索条件，列出符合的提交。用 –author 选项显示指定作者的提交，用 –grep 选项搜索提交说明中的关键字。（请注意，如果要得到同时满足这两个选项搜索条 件的提交，就必须用 –all-match 选项。） 如果只关心某些文件或者目录的历史提交，可以在 git log 选项的最后指定它们的路径。因为是放在最后位置上的选项，所以用两个短划线（–）隔开之前的选项和后面限定的 路径名
